## 2. 表单字段设计说明

| 发布类型 | 对应集合 | 专属字段 | 设计理由 |
|---|---|---|---|
| 二手交易 | trades | 分类、价格、成色、地点 | 学生买卖最关注"多少钱、新旧如何、在哪交易" |
| 失物招领 | lostFounds | 类型(lost/found)、分类、地点、时间 | 丢失/捡到是相反场景，需要用类型区分；时间和地点是找回关键 |
| 拼单搭子 | groupBuys | 拼单类型、目标人数、地点、截止时间 | 拼单核心是"几个人、什么时候截止"，目标人数决定还差几人 |
| 跑腿委托 | errands | 任务类型、酬劳、取件地点、送达地点、截止时间 | 跑腿本质是"从哪到哪、给多少钱"，取送双地点缺一不可 |

**通用字段设计：** 四类业务都保留了标题和描述，因为任何发布都需要一个简洁标题吸引注意、一段详细描述说明需求。标题限制 60 字、描述限制 500 字，避免内容过长影响浏览体验。

**发布人使用固定值：** Day4 阶段发布人和联系方式使用了固定字符串 `"我"`，后续 Day5 将接入 Pinia 模拟用户状态来动态获取。

## 3. 表单校验规则设计

| 校验字段 | 适用类型 | 校验规则 | 为什么需要 |
|---|---|---|---|
| 标题 | 全部 | 必填、不能全空格 | 没有标题无法让浏览者快速了解内容 |
| 分类 | 二手交易、失物招领 | 必选 | 分类是列表页筛选的依据，不能为空 |
| 价格 | 二手交易 | 必填、>= 0 | 二手商品必须有价格，不能是负数 |
| 成色 | 二手交易 | 必选 | 买家需要了解新旧程度来判断是否值得买 |
| 任务类型 | 跑腿委托 | 必选 | 不同任务类型难度不同，接单人需要区分 |
| 酬劳 | 跑腿委托 | 必填、>= 0 | 酬劳是接单人决策的核心因素 |
| 目标人数 | 拼单搭子 | 必填、>= 2 | 拼单至少需要 2 人，1 人不成拼 |
| 地点 | 二手交易/失物招领/拼单 | 必填 | 线下交易/取物/碰面都需要地点信息 |
| 取件/送达地点 | 跑腿委托 | 必填 | 双地点是跑腿任务区别于其他业务的核心特征 |
| 截止时间 | 拼单/跑腿 | 必填 | 超过截止时间的拼单和任务就没有意义了 |
| 描述 | 全部 | 必填 | 标题给概括，描述给细节，两者互补 |

校验方式采用手动编写 `validate()` 函数，没有引入第三方校验库。这样学生能清楚看到每一条规则的判断逻辑，便于理解和修改。

## 6. 遇到的问题与解决方法

**问题 1：el-input-number 默认值导致校验误判**

el-input-number 组件在没有输入时 `v-model` 的值是 `undefined`，但组件内部会将空值处理为 `0`。这导致价格字段为空时校验通过了（因为 `0 >= 0`）。解决方法是将表单数据中价格、酬劳、目标人数的类型从 `number` 改为 `number | undefined`，初始值设为 `undefined`，校验时判断 `=== undefined || === null`。

**问题 2：类型切换后旧数据残留**

先选"二手交易"填了价格，再切换"失物招领"，价格字段虽然不显示，但表单数据对象中的 price 值还在。这个问题通过 `selectType()` 函数中调用 `resetForm()` 解决——每次切换类型都清空整个表单对象和错误信息。

**问题 3：JSON Server v1 不需要 --watch 参数**

和 Day3 遇到的一样，新版 JSON Server 直接 `json-server db.json --port 3001` 即可，`--watch` 参数已被移除。

## 7. 今日反思

Day4 让项目从"只能看"变成了"能发布"。做完之后最大的感受是：**发布表单不是简单堆几个输入框**，它涉及业务判断（不同业务需要什么字段）、数据校验（什么字段必须填、数值范围是什么）、接口对接（POST 到哪个路径、字段名是否匹配）、交互反馈（成功怎么提示、失败怎么提示、要不要跳转）等多个环节。

表单校验虽然只写了基础的必填和数值判断，但这些规则的背后是"用户输入什么数据才是合法的"这个业务问题。比如拼单的目标人数为什么至少是 2？因为 1 个人的拼单没有意义。跑腿为什么必须有取件地点和送达地点？因为这是跑腿任务和普通代办的核心区别。这些规则不是凭空想出来的，而是从业务逻辑推导出来的。

另一个收获是：**AI 生成的代码需要围绕着"教学适用性"来删减**。AI 倾向于生成看起来更高级的代码——动态表单配置、第三方校验库、复杂的状态管理——但这些对 Day4 阶段的学生来说理解成本太高。保留直观的 `v-if` 类型切换和手写校验函数，反而比花哨的架构更适合教学场景。

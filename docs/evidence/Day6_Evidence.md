## 一、交互优化概述

Day6 的核心目标是将项目从"功能能跑"提升到"用户可用易用"的阶段。主要优化了以下四个方面：

1. **状态反馈**：为所有核心列表页补充了完整的 loading / empty / error 三态处理
2. **查询筛选**：为四个核心业务页面增加了关键词搜索功能
3. **操作反馈**：完善了收藏按钮的视觉状态变化，为所有列表页面添加了收藏功能
4. **代码整理**：抽取了 3 个可复用组件，统一了页面标题和导航一致性

## 二、新增组件

| 组件文件 | 用途 | 核心 Props / Events |
|----------|------|---------------------|
| `src/components/LoadingState.vue` | 统一的加载中状态展示 | `text`（默认"正在加载..."） |
| `src/components/ErrorState.vue` | 统一的请求失败状态展示 | `message`、`retryText`；emit `retry` |
| `src/components/SearchBar.vue` | 可复用的搜索输入框 | `modelValue`、`placeholder`；支持 `v-model` |

### 设计考量

- **加载状态**组件提取为独立组件，替代了之前各页面内联的 `.loading-box` + `.spinner` 代码，减少重复
- **错误状态**组件不仅显示错误信息，还提供"重新加载"按钮，用户无需手动刷新页面
- **搜索框**组件支持 `v-model` 双向绑定，与 Vue 生态习惯一致；自带清除按钮，提升操作效率
- 三个组件均使用 `withDefaults` + `defineProps` 定义默认值，降低使用门槛

## 四、搜索筛选实现逻辑

所有列表页面的搜索筛选采用统一的架构：

```
数据加载 → computed 计算属性（两层过滤）→ 模板渲染

第一层：分类/类型筛选（点击 tab 切换 activeFilter）
第二层：关键词搜索（输入关键词后实时过滤 filteredList）
```

关键技术点：
- 搜索关键词使用 `.trim().toLowerCase()` 进行规范化
- 搜索字段覆盖标题、类型、地点、描述等多个维度
- 搜索无结果时动态切换 EmptyState 的提示文案
- 使用 `computed` 实现响应式过滤，无需手动触发

## 十、关键收获

1. **交互优化**不是美化页面，而是让用户知道当前状态。加载中、请求失败、无数据、搜索无结果——每种状态都需要明确的视觉反馈。

2. **加载状态**和**错误状态**是两个最容易被忽视但用户感知最强的状态。一个空白页面会让用户困惑"是没数据还是加载失败？"，明确的提示消除了这种困惑。

3. **组件抽取**的价值在 Day6 体现得很明显：如果不抽取 LoadingState/ErrorState/SearchBar，五个页面需要重复写 15 段相似的模板和样式代码。

4. **搜索筛选**不追求复杂——关键词 + 分类筛选的组合已经能覆盖绝大多数使用场景。过度设计（如高级搜索面板、多条件组合）反而增加用户学习成本。

5. 收藏交互的一致性很重要——四个业务页面的收藏按钮使用相同的图标、动画和交互模式，用户在任何一个页面都不需要重新学习。

6. Day6 的做法是"在已有功能上打磨"，而不是"增加新功能"。项目稳定性没有被破坏，反而因为完善了错误处理而更加健壮。

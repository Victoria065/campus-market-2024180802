# Task1 思考问题答案
1. 为什么要开发校园轻集市？
校内学生存在大量二手闲置交易、跑腿代办、学习互助需求，但线下信息分散、无统一发布渠道；校外平台面向社会人员，存在安全与信任隐患。校园轻集市专为本校学生打造，统一供需信息渠道，建立校内熟人交易信任机制，解决校内交易场景缺失的问题。
2. 系统主要解决哪些实际问题？
一是供需信息分散，学生闲置、代办需求无法高效匹配；二是线下交易无规范流程，易产生钱款、物品纠纷；三是校外平台人员混杂，校园交易存在安全风险；四是校园各类便民服务、失物招领缺少专属线上发布平台。
3. 系统包含哪些业务场景？
二手闲置物品交易、校园跑腿代办、课程学习互助、校内公告发布（失物招领、社团活动）四大核心业务场景。
4. 七天实训最终要完成什么？
完整开发一套可正常运行的校园集市Vue3前端项目，掌握Vue3+Pinia+Router完整技术栈；产出需求分析、项目规划、每日实训记录等全套规范文档；掌握Git版本管理流程，完成代码云端托管，熟悉软件工程完整开发流程。

# Task2 思考问题答案
1. 为什么采用这种目录结构？
遵循Vue前端工程化分层模块化规范，按功能拆分代码文件，职责边界清晰；便于单人维护与多人协作，后续迭代、排查bug时可快速定位对应代码；贴合Vue3生态通用最佳实践，实现业务代码与框架基础设施解耦。
2. 每个目录负责什么？
src：项目核心业务源代码目录；docs：存放实训文档、分析报告、过程记录；scripts：项目构建、自动化校验脚本；public：无需编译的全局静态资源。
src/api封装接口请求；components存放通用复用组件；router管理页面路由跳转；stores实现全局状态管理；views存放独立业务页面；App.vue为项目根组件；main.ts是程序启动入口。
3. 哪些目录需要自己开发？
src下api、components、router、stores、views目录，以及docs文档目录均需要自主开发编写；public、scripts仅少量补充修改。
4. 哪些目录属于课程基础设施？
scripts构建脚本、public基础静态资源、main.ts初始化代码、router/Pinia/axios基础配置代码，属于课程预置基础设施，无需从零搭建。

# Task3 思考问题答案
## src/main.ts 相关
1. Vue 应用如何创建？
通过`createApp(App)`生成Vue应用实例，链式调用.use()注册路由、Pinia等全局插件，最后使用.mount('#app')将应用挂载到html页面DOM节点完成创建。
2. Pinia 如何注册？
引入createPinia创建Pinia实例，调用app.use(pinia)挂载至Vue全局，完成状态管理工具注册。
3. Router 如何注册？
导入提前配置完成的路由实例router，执行app.use(router)，将路由功能注册到全局Vue实例。
4. 项目入口在哪里？
浏览器访问入口为public/index.html；代码逻辑启动入口为src/main.ts。

## src/router/index.ts 相关
1. Router 的作用是什么？
实现前端无刷新页面跳转，通过URL路径匹配对应页面组件；可配置路由守卫实现登录权限校验、页面重定向、访问拦截等控制逻辑。
2. 为什么首页能够显示？
路由配置中对根路径`/`绑定了首页页面组件，访问项目根地址时，路由自动匹配首页组件并渲染至`<router-view>`，页面正常展示。

## src/App.vue 相关
1. App.vue 在整个项目中的作用是什么？
项目全局根组件，提供统一页面外壳布局；内置`<router-view>`承载所有路由页面；统一管理全局公共样式、全局弹窗组件，所有业务页面均嵌套在该组件内渲染。

# 五、实验思考 答案
1. 为什么课程没有要求第一天立即开发页面？
第一天核心目标是熟悉项目环境、理清需求与整体架构。若直接开发页面，会因不熟悉目录规范、路由规则、技术架构写出不符合工程标准的代码，后期大量返工；先吃透项目背景与底层结构，后续开发才能统一规范、提升效率。
2. 软件工程中「理解项目」为什么比「开始编码」更重要？
编码仅为功能实现环节，需求理解、架构设计决定项目整体方向。前期理解缺失会造成功能偏离需求、代码架构混乱难以维护，项目70%以上后期维护问题均来自前期调研不足；理清项目后编码才具备正确目标。
3. AI在软件开发中应该承担什么角色？
AI属于辅助增效工具，可完成代码解析、模板生成、文档撰写、基础语法科普等重复性工作；但业务需求判断、整体项目规划、代码逻辑校验、功能落地必须由开发者自主把控，不能完全依赖AI。
